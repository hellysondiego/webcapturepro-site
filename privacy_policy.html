Privacy Policy

Last updated: March 2025  ·  Effective immediately upon installation

Summary: WebCapture PRO does not collect, transmit, or share any personal data. All data generated or stored by this extension remains exclusively on your device.

1. What We Collect

WebCapture PRO does not collect any personal information. The extension does not send any data to external servers, analytics services, or third parties of any kind.

2. Data Stored Locally on Your Device

To function correctly, the extension stores the following data only on your local device using the Chrome chrome.storage.local API:

  • User preferences — save mode, file naming settings, counter, header visibility, font sizes.
  • License information — activation key, installation date, and a locally generated machine identifier (a random hash created at install time, never transmitted externally).
  • Temporary capture data — screenshot image data stored briefly while the editor tab loads, automatically purged within 5 minutes.

None of this data leaves your device under any circumstances.

3. File Downloads

When you capture a screenshot and save it, the resulting file (PDF or PNG) is saved to your computer via the Chrome chrome.downloads API. This operation is entirely local. No copy of the file is sent anywhere.

4. Host Permissions

WebCapture PRO requests the <all_urls> host permission. This permission is required solely to inject the screen capture script into any webpage the user chooses to capture. The extension does not read, transmit, or store the content of visited pages beyond what is visible in the current browser tab at the moment of capture.

5. No Analytics or Tracking

WebCapture PRO contains no analytics, telemetry, crash reporting, usage tracking, or advertising code of any kind. There are no third-party SDKs or trackers embedded in the extension.

6. Payment Processing

License purchases are processed externally through Stripe via a payment link that opens in a new browser tab. WebCapture PRO does not handle, store, or have access to any payment or billing information. Stripe's own privacy policy applies to that transaction.

After payment, you receive a license activation key which you enter manually in the extension. This key is validated entirely on your local device using a deterministic algorithm — no server call is made during activation.

7. Permissions Justification

  • activeTab / scripting / <all_urls> — to inject the capture script into the active webpage.
  • downloads — to save captured files (PDF/PNG) to your Downloads folder.
  • storage / unlimitedStorage — to persist user preferences and temporarily hold large screenshot data locally.
  • tabs — to open the editor page and manage batch capture workflows.
  • clipboardWrite — to support the "Copy to Clipboard" capture mode.

8. Children's Privacy

WebCapture PRO is not directed at children under the age of 13. We do not knowingly collect any information from children.

9. Changes to This Policy

If we make material changes to this Privacy Policy, we will update the "Last updated" date above. Continued use of the extension after such changes constitutes your acceptance of the revised policy.
